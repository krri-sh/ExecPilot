Start in three minutes

ExecPilot lives in Chrome’s side panel, so it stays available beside the page you’re working on.

1

Open ExecPilot

Click the extension icon and pin it for faster access. The Recipes tab is your home screen.

2

Connect a provider

Open Providers, enter your API key and preferred model, then choose Save.

3

Run a recipe

Pick a recipe, add your content or file, select a provider, and press Run.

Navigate by what you want to accomplish

Choose any section to see exactly what it does and how to use it.

Build a command center around your work

Use the supporting tabs when you want consistency, live data, or a workflow of your own.

🛠️

Skills

Save reusable instructions such as data-cleaning rules, report formats, scoring rubrics, or your writing voice. Apply one before running any recipe.

🔧

MCP Servers

Connect an HTTP or HTTP+SSE server, grant origin access, test the connection, then let compatible recipes call its live tools.

✨

New Recipe

Define a name, outcome, system prompt, and user prompt. Use {{content}} and {{skill}} where inputs should be inserted.

🔑

Providers

Bring Anthropic Claude, OpenAI, OpenRouter, ZenMux, or local Ollama. Configure the model you want for each provider.

Know where your data goes

ExecPilot is designed around provider choice and explicit connections.

Provider keys

API keys are stored locally in your browser and sent to the provider you configure.

Spreadsheet code mode

The model receives headers plus a sample, while generated transformation code runs locally across the full dataset.

MCP access

Remote tool servers require an endpoint and browser permission. Test connections before use.
