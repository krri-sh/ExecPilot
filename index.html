Ask a question. Get real numbers from your own systems.

ExecPilot speaks the Model Context Protocol (MCP) — the open standard for connecting AI to tools. Point it at your internal reporting server and the AI stops guessing and starts querying.

  • Connect multiple serversHTTP or SSE transport, with bearer-token or custom-header auth. Your finance server, your CRM connector, your BI gateway — all at once.
  • Automatic tool routingAsk in plain English. The AI discovers your server's tools, picks the right one, fills in the arguments, and calls it — with a visible trace of every call.
  • Reports come back as ExcelWhen a tool returns tabular data, ExecPilot detects it and offers a one-click .xlsx download.
  • Permission-gated by designExecPilot only gains access to a server's exact origin when you explicitly approve it — Chrome prompts you, per server.

DEFENCE IN DEPTH · ON-DEVICE ENFORCEMENT

Untrusted content stays data—not authority.

ExecPilot adds host-level controls around document Q&A and MCP tool use, reducing the chance that instructions hidden in a file or tool response can redirect the model or trigger an unintended action.

🛡️

Injection screening

Extracted document passages, MCP definitions, and tool results are checked locally for instruction overrides, role impersonation, secret extraction, encoded payloads, and external-send patterns.

🚧

Quarantined evidence

High-risk document passages and MCP results are withheld from the answering model. Users see a clear warning and the reason content was constrained.

🎯

Least-privilege tools

No MCP tool is exposed automatically. Users select the exact servers and tools permitted for each run, keeping the model's action surface intentionally narrow.

✅

Exact-call approval

Every MCP invocation pauses for confirmation showing the server, tool, risk context, and exact arguments. A changed call requires a new approval.

🔎

Schema and size enforcement

Tool arguments are validated against their declared JSON schema. Definitions, arguments, responses, and model-bound results are bounded to limit malformed or oversized payloads.

🔒

Tainted-session controls

Once untrusted MCP output enters a run, subsequent proposals carry an elevated warning. Tool traces render as inert text so server-controlled markup cannot execute in the extension UI.

Designed for enterprise review: security controls run in the extension, require no additional Chrome permission, and are available regardless of plan. Prompt injection cannot be eliminated by any LLM product, so ExecPilot combines model instructions with deterministic screening, least privilege, validation, quarantine, and human approval.

Setting up local Ollama (fixing CORS)

Browsers block extensions from calling local services unless the service allows it. Ollama needs one environment variable — OLLAMA_ORIGINS — to accept requests from ExecPilot. Pick the option that suits you.

Sets the variable only for the current PowerShell window. Ollama accepts extension requests until you close that window or stop the server. Good for trying things out.

# Stop Ollama if it's already running (quit it from the system tray first), then: $env:OLLAMA_ORIGINS = "chrome-extension://*" ollama serve

Keep this PowerShell window open while you use ExecPilot. When you close it, the setting is gone.

Sets the variable permanently for your Windows account. Survives reboots. No admin rights needed. Recommended for most people.

# Run in any PowerShell window (no admin needed): [System.Environment]::SetEnvironmentVariable("OLLAMA_ORIGINS", "chrome-extension://*", "User")

Then fully restart Ollama: right-click the Ollama icon in the system tray → Quit, and start it again (or reboot). New environment variables are only picked up on a fresh start.

Sets the variable system-wide for every user account on the machine. Requires an elevated (Administrator) PowerShell. Use this on shared workstations or when Ollama runs under a service account.

# Run PowerShell AS ADMINISTRATOR (right-click → Run as administrator): [System.Environment]::SetEnvironmentVariable("OLLAMA_ORIGINS", "chrome-extension://*", "Machine")

Then fully restart Ollama (tray → Quit → start again), or reboot the machine.

⚠ Security note: chrome-extension://* allows any browser extension on the machine to call your local Ollama. For a tighter setup, replace the wildcard with ExecPilot's exact extension ID, e.g. chrome-extension://abcdefghijklmnop… (find it at chrome://extensions).

Verify it worked: open ExecPilot → Providers → Ollama should list your local models when you run any recipe. If you still see a CORS or connection error, confirm Ollama was fully restarted after setting the variable.

Frequently asked questions

Do I need my own API keys? +
Yes — ExecPilot is BYOM (bring your own model). You connect Claude, OpenAI, OpenRouter, ZenMux, or a local Ollama instance with your own credentials. This means no AI usage markup and full control over cost and data flow.
What is MCP and what do I need to use it? +
The Model Context Protocol is an open standard for connecting AI to tools. If your company exposes an MCP server over HTTP (e.g. a reporting or data service), add its URL under MCP Servers, approve the permission prompt, and the AI can call its tools when you ask questions. Local stdio-based MCP servers are supported in the Windows desktop app.
Can I use this without sending data to the cloud? +
Yes. Connect Ollama running locally and route any recipe to the Local tier. Files are parsed on-device before anything is sent anywhere, and local-tier recipes never leave your machine. Tool-calling also works with local models that support it (Llama 3.1+, Qwen2.5, Mistral Nemo).
What file types can I upload? +
XLSX, CSV, TXT, and MD today — with PDF, DOCX and images on the roadmap. Spreadsheets can be transformed and downloaded back as a real .xlsx file using code-execution mode, and MCP report results export to Excel too.
Can I build my own recipes? +
Yes — the Recipe Builder lets you define inputs and prompts without writing code. Pro adds output contracts, representative test cases, expected-term checks, automatic version history, and version restoration. Reusable Skills remain available for writing voice, formats, rubrics, and data-cleaning rules.
How does ExecPilot keep automated workflows under control? +
Recipe chains can pause between steps for approval. Consequential MCP tools—including create, update, delete, send, publish, approve, or payment-style actions—show the server, tool, and arguments and require explicit confirmation before execution.
Can I verify AI answers and spreadsheet changes? +
Yes. Evidence mode adds clickable source references to supported text outputs. Spreadsheet code mode provides a before/after review with changed-cell and row counts; download remains locked until the review is approved.
What can I export? +
Tabular results can be downloaded as XLSX. Pro can also export the current result or a saved decision as a branded, print-ready HTML board memo, decision paper, or presentation outline.
Ollama says connection refused / CORS error — what do I do? +
Ollama needs the OLLAMA_ORIGINS environment variable set so it accepts requests from browser extensions. See the "Ollama Setup" section above for per-session, per-user, and machine-wide PowerShell commands. Remember to fully restart Ollama afterwards.
